# あなたは今、どちらですか？

<br>

| 🔨 構築できる | 🛡️ 運用できる |
|:---:|:---:|
| EC2を立ち上げられる | 障害をすぐ検知できる |
| S3にファイルを置ける | 原因を素早く特定できる |
| RDSを設定できる | コストを最適化できる |
| デプロイが完了する | SLOを定義・管理できる |

<br>

> 💡 **AWSを「構築できる」と「運用できる」は、全く別のスキルです**

# このコースの対象者

<br>

## ✅ こんなエンジニアに最適です

- AWSコンソールにログインして操作したことがある
- EC2・S3・RDSなど、主要サービスは知っている
- **でも…本番運用に自信がない**
- **でも…障害対応の経験が少なくて不安**
- **でも…チームのAWS運用ルールを整えたい**

<br>

## ❌ こんな方には難しいかもしれません
- AWSにまだ一度もログインしたことがない方

# このコースで学ぶ「5つの柱」

<br>

```
┌─────────────────────────────────────────┐
│                                         │
│  👁️  監視設計        ── 何を見るかを決める  │

│  📊  SLO管理         ── 品質基準を定義する  │
│  🚨  インシデント対応  ── 障害を素早く収束させる │
│  💰  コスト最適化     ── 無駄なお金を削る    │
│                                         │
└─────────────────────────────────────────┘
```

<br>

> これら5つを体系的に学ぶのが、このコースです

# コース修了後にできること

<br>

## 🎯 具体的なアウトプットイメージ

1. **CloudWatch** でダッシュボードを自分で設計できる
   → システムの「健康状態」を一目で把握

2. **SLO（サービスレベル目標）** を設定できる
   → 「どこまで落ちていいか」を数値で語れる

3. **インシデントの初動対応** ができる
   → 障害発生から原因特定まで迷わず動ける

<br>

> 💪 **「なんとなく不安」が「根拠ある自信」に変わります**
