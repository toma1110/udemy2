# Section 1 Lecture 3 台本

### Title

基本構文: fields/filter/sort/limit

## Slide 1

### Slide Title

基本構文

### Slide Message

表示、絞り込み、並べ替え、件数制限

### Narration

このレクチャーでは、ログインサイトの基本構文を扱います。最初に覚えるのは、フィールズ、フィルター、ソート、リミットです。表示する列を選び、条件で絞り、新しい順に並べ、件数を抑える。この4つだけで、直近ログの確認と基本的なエラー検索は始められます。複雑な関数より先に、読みやすい結果を作ることを優先します。この4つを読めるようになると、後続のスタッツ、パース、ディーダップも、パイプの途中に足す部品として理解しやすくなります。 この基本構文は、どの調査でも土台になります。難しい命令を覚える前に、表示する列を選び、必要な行を残し、見たい順に並べ、件数を絞る流れを体に入れます。

### Visual Notes

- Query pipeline with four commands
- fields filter sort limit

## Slide 2

### Slide Title

fields

### Slide Message

見たい列を選ぶ

### Narration

フィールズは、結果に表示する列を選ぶコマンドです。よく使うのは、アットタイムスタンプ、アットメッセージ、アットログストリーム、アットログです。ジェイソンログでは、サービス名、ステータスコード、エラータイプ、リクエストアイディーなどがフィールドとして見える場合があります。まず必要な列だけに絞ると、ログの横幅が減り、画面で比較しやすくなります。 表示する列を減らすと、結果の読みやすさが大きく変わります。最初は時刻、メッセージ、レベル、リクエストアイディーなど、判断に使う列だけを残すと迷いにくくなります。

### Visual Notes

- Select columns from log event
- @timestamp, @message, @logStream

## Slide 3

### Slide Title

filter

### Slide Message

必要な行だけに絞る

### Narration

フィルターは、必要なログだけに絞るコマンドです。レベルがエラー、ステータスコードが500以上、メッセージにタイムアウトを含む、といった条件を置きます。フィールドインデックスがある場合は、等号やインを使う検索でスキャン量を減らせる場合があります。一方で、ライクによる文字列検索は広くスキャンしやすいので、時間範囲を短くして使います。条件を増やしすぎると、逆に必要なログを落とすことがあります。まず広めに拾い、結果を見ながら絞るのが安全です。 フィルターは強い条件から入れすぎないことも大切です。まず広めに拾い、件数やメッセージを見てから条件を足すと、見落としを減らせます。

### Visual Notes

- Many rows filtered into error rows
- ERROR, statusCode >= 500, timeout

## Slide 4

### Slide Title

sortとlimit

### Slide Message

新しい順に、小さく見る

### Narration

ソートは並べ替えです。障害調査の入口では、アットタイムスタンプの降順で新しいログから見ます。リミットは件数制限です。最初は50件や100件に抑えて、条件が合っているか確認します。逆に、1つのリクエストを追跡する時は、古い順に並べることもあります。目的に応じて、降順と昇順を使い分けるのがポイントです。

### Visual Notes

- Latest logs first
- limit 50 as guardrail

## Slide 5

### Slide Title

pipeでつなぐ

### Slide Message

クエリは左から順に処理される

### Narration

ログインサイトのクエリは、パイプで左から右へつなぎます。最初に表示列を選び、次に条件で絞り、最後に並べ替えや件数制限を置く、という流れで読みます。途中にコメントを入れられるため、チームで共有するクエリには、何を確認するクエリなのかを短く残すと便利です。クエリを部品として読む習慣を付けると、他人のクエリも理解しやすくなります。

### Visual Notes

- Pipe character between command blocks
- Left to right processing

## Slide 6

### Slide Title

最初のクエリ

### Slide Message

直近ログを見る型を持つ

### Narration

最初の型は、アットタイムスタンプ、アットメッセージ、アットログストリームを表示し、アットタイムスタンプの降順にして、リミットを付ける形です。この型で、ログが出ているか、時刻が合っているか、メッセージにどんな情報が入っているかを確認します。ここでフィールド名が見えたら、次のレクチャー以降のエラー検索、集計、追跡へつなげます。この型は、障害時だけでなく、デプロイ後の確認や、問い合わせ調査の最初の一手としても使えます。 直近ログを見る型は、すべての調査の入り口です。何かがおかしいと感じたら、まずこの型でログが届いているか、期待した時間帯にイベントがあるかを確認します。

- Result table preview
