# Section 1 Lecture 2 台本

### Title

ロググループと時間範囲

## Slide 1

### Slide Title

安全運転

### Slide Message

まずスキャン量を小さくする

### Narration

このレクチャーでは、ログインサイトを安全に使う前提を整理します。最初に意識するのは、スキャン量です。対象ロググループ、時間範囲、結果件数、実行中クエリの停止。この4つを確認するだけで、不要な料金や長い待ち時間をかなり減らせます。障害時ほど焦りますが、まず小さく始めることが、結果として一番速い調査になります。特に個人学習や検証用アカウントでは、調査よりも先に安全策を固定しておくと安心して練習できます。 ログ調査は、急いでいる時ほど範囲を広げたくなります。しかし、広げるほど結果もノイズも増え、料金面のリスクも上がります。まず小さく見ることが、早く原因へ近づく近道です。

### Visual Notes

- Safety dashboard for query scan volume
- Narrow query funnel

## Slide 2

### Slide Title

ロググループ

### Slide Message

調査対象のログだけを選ぶ

### Narration

ロググループは、ログのまとまりです。アプリケーション、ラムダ、APIゲートウェイ、コンテナなど、サービスごとに分かれていることが多いです。複数のロググループを選ぶほど、見る範囲は広がります。最初は、アラームやダッシュボードで異常が出たサービスに近いログだけを選びます。関係ないログを外すことは、コスト削減だけでなく、ノイズ削減にもつながります。 複数のアプリや環境が同じアカウントにある場合、本当に必要なロググループだけを選ぶことが大切です。開発、検証、本番を混ぜてしまうと、同じエラーでも意味が変わります。

### Visual Notes

- Many log groups, only a few selected
- Application, API, Lambda labels

## Slide 3

### Slide Title

時間範囲

### Slide Message

直近5分から15分で始める

### Narration

時間範囲は、クエリの速度と料金に直結します。最初から1日や1週間を見るのではなく、障害が起きた時刻の前後、たとえば直近5分から15分で始めます。その範囲でログが少なすぎる、または異常の始まりが見えない場合だけ、30分、1時間と広げます。時間を広げる時も、どの仮説を確認したいのかを言葉にしてから実行します。時間を広げる時は、単に長くするのではなく、障害発生前、発生中、回復後のどこを見たいのかを分けて考えます。 障害時は、アラームが鳴った時刻と、実際に影響が始まった時刻がずれることがあります。最初は短く見て、足りなければ前後に広げる、という順番にすると判断がしやすくなります。

### Visual Notes

- Timeline with small selected window
- 5m, 15m, then expand if needed

## Slide 4

### Slide Title

limitを付ける

### Slide Message

最初は結果件数を抑える

### Narration

直近ログを見る時は、必ずリミットを付けます。最初から大量の結果を出しても、人間は読み切れません。まず50件や100件に抑えて、ログ形式、時刻、フィールド名、検索条件が合っているかを確認します。条件が正しいと分かってから、集計や深掘りへ進みます。リミットは答えを減らすためではなく、最初の確認を速くするための安全装置です。

### Visual Notes

- Query block with limit 50
- Small result table

## Slide 5

### Slide Title

キャンセル

### Slide Message

不要な実行中クエリを止める

### Narration

クエリは、画面を閉じても完了まで動き続けることがあります。不要になったクエリはキャンセルします。特に広い時間範囲、複数ロググループ、ダッシュボードの自動更新は、気づかないうちに繰り返し実行される可能性があります。調査が終わったら、実行中クエリが残っていないか、ダッシュボードにログインサイトの結果を高頻度更新で置いていないかを確認します。 実行したクエリを放置すると、不要なスキャンが続くことがあります。結果が明らかに違う、時間範囲を間違えた、ロググループを間違えたと気づいたら、すぐ止める運用にします。

### Visual Notes

- Running query with cancel button
- Stop unused query

## Slide 6

### Slide Title

チェックリスト

### Slide Message

対象、時間、件数、停止を確認する

### Narration

実行前のチェックリストは、対象、時間、件数、停止です。対象ロググループは最小か。時間範囲は短いか。リミットを付けたか。不要なクエリを止められるか。この4点を確認してから実行します。慣れてくると地味に見えますが、実務ではこの基本が一番効きます。障害対応では、派手なクエリより、安全に繰り返せる小さなクエリが役に立ちます。このチェックリストは、毎回声に出して確認するくらいで十分です。慣れるまでは、速さより安全な反復を優先します。

### Visual Notes

- Four-item checklist
- Log group, time range, limit, cancel
