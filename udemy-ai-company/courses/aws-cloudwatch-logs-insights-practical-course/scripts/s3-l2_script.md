# Section 3 Lecture 2 台本

### Title

JOIN/subquery/SOURCEの入口

## Slide 1

### Slide Title

複数ログを見る

### Slide Message

必要な時だけ範囲を広げる

### Narration

最後のレクチャーでは、ジョイン、サブクエリ、ソースを扱います。どれも複数ログや広い範囲を扱いやすくする機能です。2026年にジョインとサブクエリが加わり、複数ロググループの相関を1つのクエリで扱いやすくなりました。ただし、スキャン範囲が広がりやすいので、基本は小さく見て、必要な時だけ範囲を広げます。基本構文に慣れてから使うと、どこで範囲が広がり、どこで結果が絞られるのかを追いやすくなります。 複数ログを見るのは、最初から行う作業ではありません。単一ログで仮説を作り、関連するログに広げる理由ができた時だけ対象を増やします。

## Slide 2

### Slide Title

JOIN

### Slide Message

共通キーでログをつなぐ

### Narration

ジョインは、複数のログを共通キーでつなぐ機能です。たとえばAPIログとラムダログを、リクエストアイディーで結びます。API側では五百番台が出ていて、裏側のラムダでは例外が出ている、という関係を同じ結果で見たい時に使います。前提として、両方のログに共通キーが必要です。ログ設計の段階で、相関できるアイディーを残しておくことが重要です。 ジョインは、共通キーがそろっている時に力を発揮します。キーが欠けていたり、意味が環境ごとに違ったりすると、つながった結果を誤って解釈する可能性があります。

- API log joined with Lambda log by requestId
- One combined result row

## Slide 3

### Slide Title

subquery

### Slide Message

内側の結果を外側の条件に使う

### Narration

サブクエリは、内側のクエリで見つけたあたいを、外側のクエリの条件に使います。たとえば、下流サービスでタイムアウトしたリクエストアイディーだけを、上流サービスのログで追う、という使い方です。先に問題候補を小さく抽出し、その結果を使って次の範囲を見るための機能です。便利ですが、読みづらくなりやすいので、まず内側のクエリ単体で結果を確認します。サブクエリが複雑になったら、内側の結果を一度別に確認し、期待したキーだけが返っているかを見ます。 サブクエリは、内側で見つけた対象だけを外側で詳しく見る時に使います。便利ですが、まず内側の結果を単独で確認し、期待した値だけが返っているかを見ます。

### Visual Notes

- Inner query result feeds outer query
- Downstream failure to upstream context

## Slide 4

### Slide Title

SOURCE

### Slide Message

CLI/APIでロググループを指定する

### Narration

ソースは、シーエルアイやAPIでクエリを開始する時に、ロググループを指定するコマンドです。名前のプレフィックス、アカウント、ログクラス、タグなどで対象を選べます。コンソールで毎回ロググループを選ぶ使い方ではなく、スクリプトや運用ツールから定型調査を実行する場面に向いています。コンソール向けではない点を明確に分けて覚えます。

### Visual Notes

- SOURCE selects log groups by prefix and tags
- CLI/API label

## Slide 5

### Slide Title

制約を読む

### Slide Message

上級機能は条件と上限を確認する

### Narration

上級機能を使う時は、必ず制約を読みます。ログクラス、対応リージョン、コンソールとシーエルアイの違い、クエリの上限、対象ロググループ、スキャン量を確認します。新しい機能ほど便利に見えますが、実務で大切なのは、失敗した時に切り戻せることです。まず基本クエリで同じ仮説を小さく確認し、必要になった時だけジョインやサブクエリへ進みます。 上級機能を使う時は、対応するログクラス、リージョン、実行場所、クエリの上限を確認します。使えない時に備えて、基本クエリで同じ仮説を確認する戻り道も持ちます。

### Visual Notes

- Limitations checklist
- Key, time, cost, scope

## Slide 6

### Slide Title

まとめ

### Slide Message

小さく見て、必要な時だけ広げる

### Narration

このコースのまとめです。ログインサイトでは、小さく見て、必要な時だけ広げます。直近ログを見る。エラーで絞る。スタッツとビンで傾向を見る。パースとリクエストアイディーで追跡する。必要な時にパターン、アノマリー、ジョイン、サブクエリ、ソースを使う。この順番を持っておけば、障害時に毎回ゼロから悩まず、調査を前へ進められます。講座で扱ったクエリは、完成品ではなく出発点です。自分のログ名、フィールド名、運用ルールへ合わせて育ててください。 最後に大切なのは、クエリを増やすことではなく、判断の型を残すことです。次の障害で同じ迷いを減らせるように、自分の環境向けの定型クエリへ育てます。
